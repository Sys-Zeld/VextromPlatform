Release Notes

Resumo consolidado das ultimas revisoes aplicadas no workspace.

Escopo: manutencao administrativa, seguranca de rotas, autenticacao/autorizacao, usuarios admin e operacao de backup/restore.

Seguranca reforcada Usuarios admin no banco Manutencao revisada

Resumo executivo

  • Painel de manutencao admin criado em /admin/maintenance, com comandos operacionais de banco, API keys, tokens e sessoes.
  • Catalogo de backups implementado via tabela backup_files e servico dedicado para sincronizar, listar, restaurar, baixar e deletar backups.
  • Restore ampliado: restore do ultimo backup, restore selecionado, import de .sql via UI (limite 50MB) e opcao --no-clean.
  • Seguranca de documentos reforcada: download agora por rota controlada (token/admin), com links internos no formulario/review.
  • Gestao de usuarios admin adicionais agora persistida no banco em admin_users, com password_hash, salt, CRUD pela area de manutencao e migracao automatica do legado.

Mudancas funcionais (por modulo)

  • Manutencao administrativa (src/app.js, src/views/admin-maintenance.ejs): comandos para db_reseed, db_seed_default, db_backup, db_restore, db_restore_selected, db_delete_selected, api_key_list, api_key_create, api_key_delete, token_set_sent, token_set_draft, admin_sessions_clear.
  • Import de SQL pela UI (/admin/maintenance/restore-import): upload raw, validacao de extensao .sql, limite de tamanho, salvamento em dados/backups e execucao de restore. O parser bruto foi reposicionado para rodar somente apos autenticacao admin e autorizacao de manutencao.
  • Backups em catalogo (src/services/backups.js, src/db/migrate.js): tabela backup_files, upsert/list/get/delete, auto-recovery para erro 42P01 (tabela ausente apos restore).
  • Fluxo backup/restore em scripts: scripts/backup-database.js passou a registrar backup no catalogo e a excluir dados de public.backup_files no dump; scripts/restore-database.js passou a suportar arquivo alvo, selecao robusta do ultimo backup por timestamp e opcao --no-clean.
  • CLI Postgres cross-platform (scripts/utils/postgres-cli.js): resolve pg_dump/psql por PG_BIN_DIR, fallback para PATH e mensagens de ajuda para Windows/Linux Bitnami.
  • Admin users (src/services/adminUsers.js, src/db/migrate.js, src/app.js): CRUD de usuarios admin no banco, hash de senha com scrypt, armazenamento de salt e password_hash, suporte a exclusao de usuario e migracao inicial a partir de dados/admin-users.json quando necessario.
  • Username admin (src/services/adminUsers.js, src/app.js, src/views/admin-maintenance.ejs): o campo passou a aceitar e preservar maiusculas, minusculas e caracteres especiais, sem forcar conversao para lowercase.
  • API Keys (src/services/apiKeys.js, scripts/api-key-delete.js, package.json): suporte a delete permanente de chave e comando NPM associado.
  • Status de token (scripts/token-set-sent.js, scripts/token-set-draft.js): operacao por --token, incluindo novo comando para voltar a draft.
  • Documentos e downloads (src/services/documents.js, src/app.js, src/views/section.ejs, src/views/review.ejs): links de download internos e rotas /form/:token/documents/:id/download e /admin/documents/:id/download.
  • PDF (src/services/pdf.js): inclui tabela de documentos anexados no PDF final e mensagens relacionadas no i18n.
  • Perfis (scripts/seed-profile-bateria-smc.js, src/services/profiles.js, package.json): novo seed de perfil "Bateria SMC" e ajuste no fluxo de perfil default.
  • UI Admin Tokens (src/views/admin-tokens.ejs): botao de acesso ao painel de manutencao para perfil autorizado.
  • Respostas HTTP padronizadas (src/app.js, src/i18n/index.js): mensagens de erro do sistema alinhadas por status 400, 401, 403, 404, 409, 422, 429 e 500.
  • Configuracao (src/config/env.js, .env.example): chaves para sessoes/admin legados e ajuste recomendado de PG_BIN_DIR para Bitnami Ubuntu.

Revisoes de seguranca

  • Documentos privados: retirada de dependencia de URL publica de arquivo e concentracao do acesso em rotas controladas por autenticacao ou token valido.
  • Rate limit publico: o identificador de IP deixou de confiar em X-Forwarded-For bruto para reduzir bypass em criacao publica de token.
  • CSRF: cookie de protecao endurecido com atributos explicitos de seguranca.
  • Erros internos: respostas 5xx e erros sensiveis deixaram de expor mensagens internas do backend ao cliente.
  • Manutencao: confirmada a restricao por login admin e whitelist de comandos sem execucao arbitraria de bash pelo usuario final.
  • APIs administrativas: endpoints sensiveis mantidos sob sessao admin; rotas com API key generica foram restringidas quando nao havia ownership real implementado.

Impactos operacionais

  • Maior autonomia operacional via painel de manutencao sem depender de shell para tarefas comuns.
  • Recuperacao de backup mais previsivel (catalogo + restore selecionado + import .sql).
  • Melhoria de seguranca no acesso a anexos (download controlado por rota/autorizacao).
  • Usuarios admin adicionais agora ficam centralizados no banco em vez de arquivo JSON editavel em disco.
  • Existe mudanca de comportamento no restore padrao: limpeza do schema public antes do restore, salvo uso de --no-clean.
  • O arquivo legado dados/admin-users.json ainda pode existir como origem de bootstrap inicial, mas deixou de ser a fonte primaria apos a migracao para admin_users.
  • A validacao em ambiente local mostrou parsing OK, mas a execucao de db:migrate nao foi concluida neste workspace por bloqueio local de ambiente/Node.
