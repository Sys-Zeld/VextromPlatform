Instalacao do APP-FORM-UPS-IEC em Ubuntu

Guia completo para ambiente Linux Ubuntu com Node.js, PostgreSQL e proxy reverso (Apache ou Nginx).

1) Atualizar servidor

sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git ca-certificates gnupg lsb-release

2) Instalar Node.js e npm

Exemplo com Node 22 LTS (npm ja vem junto).

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs build-essential

Teste

node -v
npm -v

3) Instalar PostgreSQL

sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable postgresql
sudo systemctl start postgresql

Teste

sudo -u postgres psql -c "SELECT version();"

4) Criar usuario e banco exclusivos da aplicacao

Troque os valores de senha e nomes conforme seu ambiente.

sudo -u postgres psql

CREATE ROLE formapp_user WITH LOGIN PASSWORD 'SENHA_FORTE_AQUI';
CREATE DATABASE formupsiec OWNER formapp_user;
GRANT ALL PRIVILEGES ON DATABASE formupsiec TO formapp_user;
\q

Teste de conexao

psql "postgres://formapp_user:SENHA_FORTE_AQUI@127.0.0.1:5432/formupsiec" -c "SELECT current_user, current_database();"

5) Criar pasta da aplicacao com permissoes corretas (Node + Apache)

Padrao recomendado: dono deploy, grupo www-data (Apache/Nginx).

sudo mkdir -p /var/www/formulario
sudo adduser --disabled-password --gecos "" deploy
sudo usermod -aG www-data deploy
sudo chown -R deploy:www-data /var/www/formulario
sudo chmod -R 2750 /var/www/formulario

Se o processo Node rodar com outro usuario, adicione esse usuario no grupo www-data.

6) Publicar codigo e configurar .env

cd /var/www/formulario
sudo -u deploy git clone SEU_REPO_AQUI .
cp .env.example .env
nano .env

Exemplo minimo de variaveis:

NODE_ENV=production
PORT=3000
APP_BASE_URL=https://seu-dominio.com
DATABASE_URL=postgres://formapp_user:SENHA_FORTE_AQUI@127.0.0.1:5432/formupsiec
DATABASE_SSL=false
ADMIN_USER=admin
ADMIN_PASS=troque-essa-senha
ADMIN_SESSION_SECRET=troque-para-um-segredo-longo
API_KEY_PEPPER=troque-para-um-segredo-diferente

7) Instalar dependencias e preparar banco

cd /var/www/formulario
sudo -u deploy npm install
sudo -u deploy npm run db:migrate
sudo -u deploy npm run db:seed

Teste rapido da app

sudo -u deploy npm run start
# em outro terminal:
curl -I http://127.0.0.1:3000

8) Rodar em modo desenvolvimento ou producao

Desenvolvimento

cd /var/www/formulario
sudo -u deploy npm run dev

Producao (PM2 + ecosystem.config.js)

cd /var/www/formulario
sudo npm install -g pm2
sudo -u deploy pm2 start ecosystem.config.js --env production
sudo -u deploy pm2 save
sudo pm2 startup systemd -u deploy --hp /home/deploy

9) Configurar proxy reverso - Apache

sudo a2enmod proxy proxy_http proxy_wstunnel headers rewrite ssl
sudo nano /etc/apache2/sites-available/formulario.conf
<VirtualHost *:80>
  ServerName seu-dominio.com
  ProxyPreserveHost On
  ProxyPass / http://127.0.0.1:3000/
  ProxyPassReverse / http://127.0.0.1:3000/
  RequestHeader set X-Forwarded-Proto "http"
</VirtualHost>
sudo a2ensite formulario.conf
sudo apache2ctl configtest
sudo systemctl reload apache2

10) Configurar proxy reverso - Nginx (alternativa)

sudo apt install -y nginx
sudo nano /etc/nginx/sites-available/formulario
server {
  listen 80;
  server_name seu-dominio.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
sudo ln -s /etc/nginx/sites-available/formulario /etc/nginx/sites-enabled/formulario
sudo nginx -t
sudo systemctl reload nginx

11) Checklist final

  • node -v e npm -v ok
  • psql ... -c "SELECT 1" ok
  • npm run db:migrate e npm run db:seed ok
  • App respondendo em http://127.0.0.1:3000
  • Dominio respondendo via Apache ou Nginx

Instalacao concluida.
