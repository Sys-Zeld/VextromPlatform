Documentacao de API

Projeto: APP-FORM-UPS-IEC

Fonte: src/app.js | Atualizado em 2026-03-07

Auth por API Key (Bearer) Escopos por endpoint Fallback: sessao admin valida CSRF em rotas web/form JSON para endpoints REST

0) Autenticacao da API

Para consumir os endpoints REST, envie uma chave via header:

Authorization: Bearer <API_KEY>
ou
X-API-Key: <API_KEY>

Comandos de gestao:

npm run api:key:create -- --name "integracao-x" --scopes "fields:read,spec:read,spec:write"
npm run api:key:list
npm run api:key:revoke -- 1

1) Endpoints REST (JSON)

Metodo Rota Auth Descricao
GET/fieldsAPI Key (fields:read)Lista campos. Filtro opcional: ?section=....
POST/fieldsAPI Key (fields:write)Cria campo.
PUT/fields/:idAPI Key (fields:write)Atualiza campo.
DELETE/fields/:idAPI Key (fields:write)Exclui campo.
GET/equipment/:id/specificationAPI Key (spec:read)Busca especificacao de um equipamento.
PUT/equipment/:id/specificationAPI Key (spec:write)Salva especificacao via values.
POST/form/:token/docs/uploadCom token + CSRFUpload de PDF (raw body).

2) Rotas de formulario/export

Metodo Rota Descricao
GET/POST/form/:token/specificationTela e salvamento da especificacao.
GET/form/:token/reviewPagina de revisao.
POST/form/:token/send-emailEnvia submissao por email.
GET/form/:token/pdfBaixa PDF da submissao.
GET/form/:token/export.jsonBaixa JSON consolidado.
GET/POST/form/public/:draftToken/specificationFluxo publico para criar token/submissao.
